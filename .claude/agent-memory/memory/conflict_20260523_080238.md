# conflict_20260523_080238

## Context

Merging `ap-susp-update` into `remote-suspend` conflicted in `packages/backend/src/core/UserSuspendService.ts`.

## Resolution

`ap-susp-update` delivered suspend/unsuspend activities directly through `ApDeliverManagerService` and now awaited `manager.execute()`.
`remote-suspend` replaced that direct delivery with `AccountUpdateService.publishToFollowersAndSharedInboxAndRelays`, which also sends the account update to followers, known shared inboxes, and relays.

Kept the `remote-suspend` delivery path and applied the async error-propagation intent from `ap-susp-update` by awaiting `publishToFollowersAndSharedInboxAndRelays`.
